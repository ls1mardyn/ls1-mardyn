534 *
535 * In library mode, this function is used to externally control the simulation. In normal usage, this function is

541 /*** @brief Performs necessary setup before the first timestep, which includes initializing all timers. */
542 void preSimLoopSteps();
543
544 /*** @brief Performs immediate cleanup after the simulation ends.
545 *
546 * The final checkpoint is written, the final plugin call is made, and all timers are stopped.
547 * Relevant timer information is printed.
548 */
549 void postSimLoopSteps();
550
551 /*** @brief Used to flag simulation as finished and to proceed to cleanup. Only used when ls1 is compiled and used
552 * as a library, and not in normal usage.
553 *
554 * Normally, the simulation is controlled entirely by the simulate() function, which determines whether the
555 * simulation is complete by checking the keepRunning() function.
556 * However in library mode, it is expected that the external code will manually control the simulation by calling
557 * simulateOneTimeStep(). As such, the keepRunning() function will never be called, and the exit conditions will
558 * never be checked on ls1's side. \n
559 * This function acts as a way for the external code to perform some cleanup steps, before ending the simulation
560 * on the ls1 side. For now, it only sets the simulationDone boolean to true, which is used as a check within
561 * the postSimLoopSteps() function.
562 * */
563 void markSimAsDone();

568 /*** @brief Act as safeguards for the preSimLoopSteps(), simulateOneTimestep() and postSimLoopSteps() functions.
569 *
570 * These three functions are public, since they need to be reachable by external code when ls1 is compiled
571 * as a library. However it is possible to call them out of order, causing unexpected behaviour. As such,
572 * these bool values help keep track of the simulation and can be used to verify the state the simulation is in.
